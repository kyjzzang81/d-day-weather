# 1. v2.0 → v2.1 변경 요약

| 영역 | v2.0 | v2.1 |
|---|---|---|
| 새로 만들기 활동 선택 | 활동 카테고리 10개 표시 | 선호 활동 우선 노출 + 전체 활동 표시 |
| 알림 권한 | D-DAY 기본 알림 정의 | 첫 D-DAY 저장 직후 권한 요청 |
| 온보딩 연계 | 미정 | 온보딩에서는 알림 요청하지 않음 |
| family_kids | 일정 옵션 | 사용자 기본 overlay가 있으면 prefill |

---

# 2. 핵심 역할

D-DAY는 사용자가 이미 정한 일정을 저장하고, 시간이 가까워질수록 날씨 정보와 행동 가이드를 단계적으로 제공하는 화면이다.

추천 받기는 DISCOVER가 담당하고, D-DAY는 직접 입력 + 저장 일정 관리가 핵심이다.

---

# 3. 새로 만들기 플로우

## 3-1. 진입점

```text
D-DAY 탭 → 새로 만들기
TODAY → 다음 일정 찾아보기 CTA
DISCOVER 결과 → D-DAY로 저장
```

## 3-2. 입력 항목

| 항목 | 필수 | 설명 |
|---|---|---|
| 일정명 | 필수 | 자동 제안 가능 |
| 날짜 | 필수 | 단일 날짜 또는 범위 |
| 도시/지역 | 필수 | 검색 또는 현재 위치 |
| 활동 카테고리 | 필수 | 10개 중 1개 |
| theme overlay | 선택 | family_kids |
| 알림 | 기본 ON | 저장 후 권한 요청 |

# 4. 활동 카테고리 선택

## 4-1. 선호 활동 설정 완료 사용자

사용자의 선호 활동을 먼저 노출한다.

```text
활동 카테고리 선택

자주 쓰는 활동
[피크닉/도시산책] [카페/맛집] [전시/문화]

전체 활동
[해변] [등산/트레킹] [캠핑] [일출/일몰] ...
```

## 4-2. 선호 활동 미설정 사용자

전체 활동만 노출한다.

```text
활동 카테고리 선택

[해변]
[등산/트레킹]
[캠핑]
[일출/일몰]
[사진/뷰]
[피크닉/도시산책]
[카페/맛집]
[축제/이벤트]
[온천/리조트]
[전시/문화]
```

## 4-3. 선호 활동 변경 링크

보조 링크로 제공 가능하다.

```text
자주 쓰는 활동을 설정하면 더 빠르게 선택할 수 있어요.
[선호 활동 설정]
```

# 5. family_kids overlay prefill

## 5-1. 사용자 기본값이 있는 경우

```text
user_preferences.default_theme_overlay = family_kids
```

이면 D-DAY 저장 확인 화면에서 `아이와 함께` 옵션을 기본 ON으로 표시한다.

## 5-2. 사용자는 일정별로 변경 가능

```text
[ ] 아이와 함께
```

을 켜거나 끌 수 있다.

# 7. 알림 권한 요청 정책

## 7-1. 온보딩에서는 요청하지 않는다

첫 실행 온보딩은 가치 소개 + 위치 설정만 수행한다.

## 7-2. 첫 D-DAY 저장 직후 요청한다

저장 완료 후, OS 권한 요청 전에 사전 안내를 표시한다.

```text
이 일정의 날씨 변화와 준비 시점을 알려드릴게요.

D-14 실제 예보 도착
D-7 짐 싸기 체크
D-1 최종 점검
당일 시간대별 안내까지 받아볼 수 있어요.

[알림 받기]
[나중에]
```

## 7-3. 이후 일정 저장 시

이미 권한이 허용되어 있으면 추가 요청하지 않는다.  
권한이 거부되어 있으면 설정 안내를 제공한다.

# 8. 시간축 단계

기존 v2.0 정의를 유지한다.

```text
D-84 → D-28 → D-14 → D-7 → D-1 → D-0
```

D-DAY Carousel에는 5단계를 노출한다.

```text
[D-84] [D-28] [D-14] [D-7] [D-1]
```

D-0은 D-1 단계의 당일 상태로 처리한다.

---

# 9. 점수 표현

D-DAY는 행동형 라벨을 사용한다.

| Score | D-DAY 표현 |
|---:|---|
| 85~100 | 강력추천 |
| 70~84 | 추천 |
| 50~69 | 보통 |
| 0~49 | 비추천 |

도트는 보조 표현으로만 사용한다.
