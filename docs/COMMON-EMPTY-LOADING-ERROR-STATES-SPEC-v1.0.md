# COMMON EMPTY / LOADING / ERROR STATES Spec (MVP)

## Version
**v1.0** — 공통 Empty / Loading / Error 상태 정의

## Document Type
Product / UX Specification

## Status
MVP Draft

---

# 1. 문서 목적

이 문서는 ggg 앱 전체에서 공통으로 사용하는 **Empty / Loading / Error / Permission / Unavailable 상태**를 정의한다.

각 화면별로 예외 상태를 따로 처리하면 사용자 경험과 문구 톤이 흔들릴 수 있다.  
따라서 MVP에서는 공통 상태 패턴을 정의하고, TODAY / DISCOVER / D-DAY / 마이페이지 / 설정 / 검색 / 위치 선택 화면에서 재사용한다.

---

# 2. 핵심 결정사항

| 항목 | 결정 |
|---|---|
| 상태 유형 | Empty / Loading / Error / Permission / Unavailable |
| Loading UI | Skeleton 우선 |
| 기술 오류 노출 | 금지 |
| 모든 상태 | 사용자가 할 수 있는 다음 행동 제공 |
| Score 계산 실패 시 D-DAY 저장 | 허용 |
| 추천 결과 부족 시 활동 전체로 보기 | 포함 |
| Empty/Error 문구 | 공통 톤 적용 |
| 권한 없음 | 대체 경로 제공 |

---

# 3. 공통 상태 분류

| 상태 유형 | 의미 | 예시 |
|---|---|---|
| Empty | 데이터가 없음 | 저장된 D-DAY 없음 |
| Loading | 데이터를 불러오는 중 | TODAY 날씨 로딩 |
| Error | 요청 또는 처리 실패 | 날씨 API 실패 |
| Permission | 권한이 없음 | 위치/알림 권한 없음 |
| Unavailable | 아직 지원 불가 또는 데이터 부족 | 특정 지역 데이터 준비 중 |

---

# 4. 공통 UI 원칙

## 4-1. 상태 설명 + 의미 + 다음 행동

모든 상태 문구는 아래 구조를 따른다.

```text
상태 설명
원인 또는 의미
다음 행동 CTA
```

나쁜 예:

```text
데이터가 없습니다.
```

좋은 예:

```text
아직 저장된 일정이 없어요.
좋은 날짜와 장소를 찾아볼까요?

[DISCOVER로 이동]
[직접 일정 만들기]
```

---

## 4-2. 기술적 원인을 직접 노출하지 않음

나쁜 예:

```text
API request failed: 500
```

좋은 예:

```text
날씨 정보를 불러오지 못했어요.
잠시 후 다시 시도해주세요.

[다시 시도]
```

---

## 4-3. 복구 가능한 상태와 불가능한 상태 구분

| 상태 | CTA |
|---|---|
| 네트워크 오류 | 다시 시도 |
| 위치 권한 없음 | 현재 위치 사용 / 지역 직접 선택 |
| 검색 결과 없음 | 다른 검색어 입력 |
| 추천 결과 없음 | 범위 넓히기 / 다른 날짜 보기 / 활동 전체로 보기 |
| 데이터 준비 중 | 상위 지역으로 보기 / 다른 지역 선택 |
| 로그인 필요 | 로그인하기 / 나중에 |

# 6. Empty 상태

## 6-1. 저장된 D-DAY 없음

### 사용 화면

```text
D-DAY 탭
마이페이지 → 저장된 D-DAY
```

### 문구

```text
아직 저장된 일정이 없어요.
좋은 날짜와 장소를 찾아볼까요?

[DISCOVER로 이동]
[직접 일정 만들기]
```

### CTA

| CTA | 이동 |
|---|---|
| DISCOVER로 이동 | DISCOVER 첫 화면 |
| 직접 일정 만들기 | D-DAY 새로 만들기 |

---

## 6-2. 완료된 D-DAY 없음

### 사용 화면

```text
마이페이지 → 완료된 D-DAY
```

### 문구

```text
완료된 일정이 아직 없어요.
저장한 일정이 지나면 이곳에서 확인할 수 있어요.
```

### 보조 CTA

```text
[저장된 일정 보기]
```

---

## 6-3. 선호 활동 미설정

### 사용 화면

```text
TODAY 활동 섹션
```

### 문구

```text
선호 활동을 선택해주세요.
자주 하는 활동을 설정하면
오늘 어떤 활동이 좋은지 알려드릴게요.

[선호 활동 설정하기]
```

### CTA

```text
선호 활동 설정 화면
```

---

## 6-4. 최근 위치 없음

### 사용 화면

```text
위치 선택 화면
```

### 문구

```text
최근 위치가 없어요.
현재 위치를 사용하거나 지역을 직접 선택해주세요.
```

### CTA

```text
[현재 위치 사용]
[지역 직접 선택]
```

---

## 6-5. 검색 결과 없음

### 사용 화면

```text
위치 검색
DISCOVER 검색
D-DAY 지역 선택
```

### 문구

```text
검색 결과가 없어요.
다른 지역명이나 장소명으로 검색해보세요.

예: 성수동, 부산, 제주 안덕면
```

### CTA

검색 입력 상태를 유지한다.  
추천 검색어를 보조로 노출할 수 있다.

# 7. Error 상태

## 7-1. 네트워크 오류

### 사용 화면

앱 전체.

### 문구

```text
인터넷 연결이 불안정해요.
연결 상태를 확인하고 다시 시도해주세요.

[다시 시도]
```

---

## 7-2. 날씨 데이터 불러오기 실패

### 사용 화면

```text
TODAY
DISCOVER
D-DAY
```

### 문구

```text
날씨 정보를 불러오지 못했어요.
잠시 후 다시 시도해주세요.

[다시 시도]
[지역 변경]
```

---

## 7-3. Score 계산 실패

### 일반 화면 문구

```text
날씨 적합도를 계산하지 못했어요.
기본 날씨 정보는 확인할 수 있어요.

[다시 계산]
```

### D-DAY 저장 시

Score 계산에 실패해도 D-DAY 저장은 허용한다.

```text
날씨 적합도를 계산하지 못했어요.
일정은 저장할 수 있지만 점수는 나중에 업데이트돼요.

[그래도 저장]
[다시 시도]
```

### 결정 이유

- 사용자의 1차 목적은 일정 저장일 수 있다.
- 점수는 나중에 업데이트 가능하다.
- 저장 실패보다 점수 지연이 사용자에게 덜 치명적이다.

# 8. Permission 상태

## 8-1. 위치 권한 없음

### 사용 화면

```text
온보딩 Step 2
TODAY
DISCOVER 내 주변
위치 설정
```

### 문구

```text
현재 위치를 사용할 수 없어요.
지역을 선택하면 오늘 날씨와 추천을 볼 수 있어요.

[현재 위치 사용]
[지역 직접 선택]
```

---

## 8-2. 위치 정확도 낮음

### 문구

```text
위치를 대략적으로 확인했어요.
더 정확한 동네 날씨를 보려면 위치 정확도를 높여주세요.

[위치 정확도 높이기]
[이 위치로 계속]
```

---

## 8-3. 알림 권한 없음

### 사용 화면

```text
알림 설정
D-DAY 저장 후 알림 안내
D-DAY 상세 알림 관리
```

### 문구

```text
알림 권한이 꺼져 있어요.
D-DAY 알림을 받으려면 기기 설정에서 알림을 켜주세요.

[기기 설정으로 이동]
```

---

## 8-4. 로그인 필요

### 사용 조건

```text
클라우드 동기화
여러 기기에서 D-DAY 확인
계정 관리
계정 삭제
```

### 문구

```text
로그인이 필요해요.
D-DAY를 안전하게 저장하고 여러 기기에서 확인할 수 있어요.

[Google로 계속하기]
[Apple로 계속하기]
[Kakao로 계속하기]
[나중에]
```

# 9. Unavailable 상태

## 9-1. 지역 데이터 준비 중

### 문구

```text
아직 이 지역의 데이터를 준비 중이에요.
가까운 상위 지역 기준으로 볼 수 있어요.

[상위 지역으로 보기]
[다른 지역 선택]
```

### 예시

```text
성수동 데이터 준비 중
→ 성동구로 보기
```

---

## 9-2. Weather grid 매핑 실패

### 문구

```text
날씨 데이터를 연결하지 못했어요.
가까운 지역 기준으로 볼까요?

[가까운 지역으로 보기]
[다른 지역 선택]
```

---

## 9-3. 추천 결과 부족

### 사용 화면

```text
DISCOVER P0
DISCOVER P2
DISCOVER P3
```

### 문구

```text
조건에 맞는 추천이 부족해요.
조건을 조금 넓혀볼까요?

[범위 넓히기]
[다른 날짜 보기]
[활동 전체로 보기]
```

### 결정 이유

- 선호 활동이나 특정 활동 조건이 너무 좁을 수 있다.
- DISCOVER의 탐색성을 유지할 수 있다.
- 사용자가 같은 화면 안에서 조건을 완화할 수 있다.

# 10. 화면별 대표 상태 매핑

## 10-1. TODAY

| 상태 | 처리 |
|---|---|
| 위치 권한 없음 | 지역 직접 선택 유도 |
| 날씨 로딩 | Hero/지표 skeleton |
| 날씨 실패 | 다시 시도 + 지역 변경 |
| 선호 활동 없음 | 선호 활동 설정 CTA |
| 활동 추천 실패 | Hero는 유지, 활동 추천만 에러 |
| D-DAY 없음 | D-DAY 섹션 숨김 |

---

## 10-2. DISCOVER

| 상태 | 처리 |
|---|---|
| 검색 결과 없음 | 검색어 변경 유도 |
| 추천 결과 없음 | 범위/날짜/활동 변경 CTA |
| score 계산 실패 | 추천 카드 비활성 또는 재시도 |
| 위치 권한 없음 | 지역 직접 선택 |
| 전국 후보 부족 | 운영 후보 풀 확장 필요 안내 |

---

## 10-3. D-DAY

| 상태 | 처리 |
|---|---|
| 저장 일정 없음 | DISCOVER / 직접 만들기 CTA |
| 일정 로딩 | D-DAY 카드 skeleton |
| score 실패 | 일정 유지, 점수 나중 업데이트 |
| 저장 실패 | 다시 시도 |
| 알림 권한 없음 | 설정 이동 CTA |
| 완료 일정 없음 | 안내 문구 |

---

## 10-4. 마이페이지 / 설정

| 상태 | 처리 |
|---|---|
| 로그인 전 | 로그인 CTA |
| 저장된 D-DAY 없음 | DISCOVER / 직접 만들기 CTA |
| 완료된 D-DAY 없음 | 안내 문구 |
| 계정 삭제 실패 | 다시 시도 |
| 설정 저장 실패 | 다시 시도 |

# 14. MVP 포함 / 제외

## 14-1. 포함

- Empty 상태 공통 문구
- Loading skeleton 기준
- Error 상태 공통 문구
- Permission 상태 공통 문구
- Unavailable 상태 공통 문구
- Score 계산 실패 시 D-DAY 저장 허용
- 추천 결과 부족 시 활동 전체로 보기
- 화면별 대표 상태 매핑
- 공통 CTA 규칙
- 공통 톤앤매너
- 기본 계측 이벤트

## 14-2. 제외

- 복잡한 오류 코드 사용자 노출
- 개발자용 오류 상세 화면
- 오프라인 모드 전체 지원

# 15. 최종 요약

```text
공통 상태 처리
→ Empty / Loading / Error / Permission / Unavailable로 분류
→ 기술 오류는 숨김
→ 항상 다음 행동 CTA 제공
→ Score 계산 실패 시 D-DAY 저장 허용
→ 추천 부족 시 조건 완화 CTA 제공
```

MVP에서는 예외 상태를 사용자의 막힘으로 두지 않고, 다음 행동으로 자연스럽게 이어지게 만드는 것을 목표로 한다.
